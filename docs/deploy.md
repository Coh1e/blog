# Deploy

The build output is a flat folder of HTML/CSS/JS at `react/dist/`. Drop it on any static host. The two we'd actually pick: **Cloudflare Pages** (CDN-edge, generous free tier) and **Vercel** (best DX).

## Build command

```bash
cd react
npm install        # postinstall reapplies the vite-react-ssg patch
npm run build      # → dist/
```

`dist/` contains:
- One HTML file per route under nested directories (`dist/index.html`, `dist/en/index.html`, `dist/posts/markdown-blog/index.html`, …)
- A single hashed JS bundle in `dist/assets/`
- A single hashed CSS bundle in `dist/assets/`
- A `static-loader-data-manifest-*.json` (used by vite-react-ssg's loader hydration; a few KB)

About **97 HTML files** at current content size. Will scale linearly with entry count.

## Cloudflare Pages

The simplest deploy.

1. **From the dashboard**: Pages → "Create a project" → "Connect to Git" → pick the repo.
2. Build settings:
   - **Framework preset**: None
   - **Build command**: `cd react && npm install && npm run build`
   - **Build output directory**: `react/dist`
   - **Root directory**: leave empty (we want the build to start from the repo root so the `cd react` works)
   - **Environment variables**: `NODE_VERSION = 20` (or 22; avoid 25.x — vite-react-ssg's CJS path on newer Node has been finicky)
3. Save & deploy.

Cloudflare auto-handles trailing-slash routing for nested-directory builds, so `/en/`, `/all/`, `/posts/markdown-blog/` all hit the right `index.html`.

For preview deployments per branch: turn on "Preview deployments" in the project settings. Each non-`main` branch gets its own URL.

### Custom domain

Cloudflare Pages → project → Custom domains → add your domain. If the domain already lives on Cloudflare DNS, this is one click. Otherwise add the CNAME they tell you to.

## Vercel

Almost as easy.

1. **From the dashboard**: New → Project → import the repo.
2. Settings:
   - **Framework Preset**: Other
   - **Root Directory**: `react`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Install Command**: `npm install`
   - **Node version**: 20 in Project → Settings → General
3. Deploy.

Vercel auto-rewrites `/foo/` → `/foo/index.html` for nested directories. Trailing slashes work.

## Local preview of the build

```bash
npm run preview     # serves dist/ on http://localhost:4173
```

Always do this once before deploying — it's the only place dev-mode oddities and prod-bundle differences surface.

## SEO files

These ship with every build:

- **`dist/robots.txt`** — copied from `public/robots.txt`. Allows all crawlers, points to `/sitemap.xml`.
- **`dist/sitemap.xml`** — one `<url>` entry per route (97 today). Each entry has `xhtml:link` alternates for both languages plus `x-default`. Generated by `scripts/generate-seo-files.mjs` as a `postbuild` step.
- **`dist/feed.xml`** (CN) and **`dist/en/feed.xml`** (EN) — RSS 2.0 with the 20 most recent posts + notes per language. Same script.

The script reads `SITE_URL` from the environment. The default `https://xie.example` is a placeholder. **You must set this in your deploy provider** or every URL in the sitemap and feed will be wrong:

- **Cloudflare Pages**: Settings → Environment variables → Production: `SITE_URL=https://your-domain`
- **Vercel**: Project → Settings → Environment Variables → `SITE_URL=https://your-domain`

The script logs a warning when the placeholder is in use, so you'll see it in build logs.

## Search (Pagefind)

Wired into the build. `npm run build` runs `pagefind --site dist --output-path dist/pagefind` after the SSG step. Pagefind crawls the rendered HTML and produces:

- `dist/pagefind/pagefind.js` — runtime bundle
- `dist/pagefind/index/` — sharded inverted index, downloaded on demand
- `dist/pagefind/fragment/` — short result snippets, downloaded on demand

The index splits by `<html lang>` automatically (we now set this per page via `PageHead`). Both `zh-cn` and `en` indexes are built. Total index size is small enough (a few hundred KB at current content volume) that there's no special CDN config needed.

The frontend integration is in `src/components/Search.tsx`. The `<SearchButton>` toggles a modal that lazy-loads `/pagefind/pagefind.js` on first open (so the search bundle isn't paid for by readers who don't use it). Custom UI matches the site's typography; results use Pagefind's `excerpt` (already keyword-highlighted with `<mark>`).

**In dev (`npm run dev`)**, the search index doesn't exist yet — clicking search will show "Search index not built. Available after a production build." Run `npm run build && npm run preview` to test search end-to-end.

## What you might add later

These are out of scope for v1; document here so future-you finds them:

- **OG images** — per-article `og:image`. Simplest path: a build-time function that renders SVG → PNG via `satori` or similar, then `<PageHead>` adds `<meta property="og:image">`. Pages won't ship without this; it just hurts shareability.
- **JSON-LD structured data** — inline `<script type="application/ld+json">` per article (Article + BreadcrumbList). Helps Google Discover.
- **Filtered search** — Pagefind supports `data-pagefind-filter` attributes on rendered HTML to enable faceted search (filter by tag, type, language). Add `<meta>` tags or wrap content sections with the attribute, then expose filter UI in `Search.tsx`.

## CI

There isn't one yet. When there is, the minimum useful step:

```yaml
- run: cd react && npm install
- run: cd react && npm run typecheck
- run: cd react && npm run build
```

Type errors and broken `import.meta.glob` resolutions show up at typecheck or build. Wikilink warnings are stderr only — don't fail CI on them (they're a soft signal).

## Troubleshooting

- **`/en/` → 404 on the deployed site**: check that `ssgOptions.dirStyle: "nested"` is still in `vite.config.ts`. Without it, vite-react-ssg writes `dist/en.html` instead of `dist/en/index.html`.
- **Patch not applied**: ensure `"postinstall": "patch-package"` is still in `package.json`. CI providers sometimes run `npm install --omit=dev` which skips devDependencies — `patch-package` is a devDep, so it would silently skip. Either move it to dependencies (the Cloudflare/Vercel default install includes both) or use `--include=dev`.
- **Hydration mismatch warning in dev**: usually a `Date` formatted differently between server and client. Check `formatDate()` and any locale-sensitive code; the SSG run uses Node, the dev server uses Node + browser interop.
- **Slow build**: vite-react-ssg's SSG step is single-threaded by default. Bump `ssgOptions.concurrency` (default 20) if needed; on a small site this rarely matters.
